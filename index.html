事件兼容整理

以前对 dom 事件兼容方面的知识一直是懵懵懂懂,今天抽空把这些知识点做了一次梳理,以免后患

event 对象

  • chrome 和 ie 都支持全局event对象,即在事件函数中可直接调用 event 对象,当然这种写法不符合 w3c标准

  • 火狐只支持标准的事件对象写法,即在事件函数中的第一个参数为事件对象,chrome 与标准 ie 也支持此种写法

所以如果需要兼容低版本 ie 就需要作兼容处理,否则可以采用标准写法

事件绑定方式

如果对象的一个事件需要绑定两个事件函数,上面的方法就不适用了,因为上面的这种写法会产生覆盖问题,此时就需要使用第二种事件绑定方法了,即 addEventListener||attachEvent

  • addEventListener:此方法是标准的事件绑定方法,使用方法为element.addEventListener(event, function, useCapture),参数列表中,第一个参数为要给element对象添加的事件类型,要注意的是事件的名字前面没有”on”,第二个参数为事件函数,第三个参数决定事件在捕获还是冒泡阶段执行,默认为 false 即事件在冒泡阶段执行,如果设为 true 则事件在捕获阶段执行.
  • attachEvent:此方法只在 ie 下可用(ie11好像不支持而回归标准addEventListener了),使用方法为attachEvent(event, listener),第一个参数为要添加的事件类型,第二个参数为事件函数,与 addEventListener 不同的是事件类型名字前需要添加”on”,比较坑爹的是事件函数中this指向的的对象为 window 所以在事件函数中需要用 call 处理 this 指向.
